Arquitectura (MVC)

Esta app sigue una arquitectura Modelo‑Vista‑Controlador para separar responsabilidades y mantener el código fácil de entender.

Modelo

  • Contact: entidad JPA que representa mensajes del formulario.
  • Member: modelo simple para mostrar integrantes.

Controlador

  • WikiController: rutas de navegación (proyecto, miembros, arquitectura).
  • ContactController: crear, listar y borrar mensajes.

Vista

  • Plantillas Thymeleaf: proyecto.html, miembros.html, arquitectura.html, formulario.html.
  • Componentes reutilizables: fragments/header y fragments/footer.

Capas de datos

El módulo de contacto usa un flujo simple:

  1. Vista envía el formulario a /formulario
  2. ContactController delega a ContactService
  3. ContactServiceImpl persiste usando ContactRepository
  4. Se renderiza la vista con el historial del buzón
